Pseudonymous identity — no email

AMP ID (hashed) is the master identity — no email collected. Sessions, devices and return visits stitch retroactively the moment a user logs in. If onboarding changes and a new ID is introduced later, the switch is non-destructive.

AMP ID master email not required user properties: tier, BU flags

Masked replay for support

Session replay is a masked reconstruction, not a recording — every sensitive element hidden by default. A support agent can ask a caller for their AMP ID and replay exactly what happened, without exposing personal data.

masked by default find session by AMP ID

BigQuery, natively

All behavioral data is queryable from BigQuery in place — no duplicated storage cost, you pay only BigQuery compute. Egress applies only if you copy data out. 13 months retention; Milk's team joins it to transactional data for modeling.

no duplicate storage 13-month retention
